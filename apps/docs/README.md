# @streamd/docs

Internal — not published. Docusaurus-powered documentation site for
the streamd toolchain. Combines hand-written guides under `docs/`
with the TypeDoc-generated API reference shipped under `static/api/`.

## Structure

- `docs/intro.md` — landing page and monorepo overview.
- `docs/packages/*.md` — one reference page per published package:
  `@streamd/parser`, `@streamd/tokens`, `@streamd/html`,
  `@streamd/react`, `@streamd/react-native`, `@streamd/plugins`,
  `@streamd/plugin-shiki`, `@streamd/cli`.
- `docs/recipes/*.md` — end-to-end guides: LLM streaming, safe
  pipeline construction, custom themes, component overrides, Shiki
  integration, math rendering, and migration paths from
  `markdown-it` / `commonmark.js` / `marked`.
- `static/api/` — TypeDoc HTML output. Generated by the parent
  repo's TypeDoc task; do not hand-edit.

Navigation is wired in `sidebars.ts`. Branding and theme config live
in `docusaurus.config.ts`.

## Run locally

```bash
npm install                              # from the repo root
npm run dev --workspace=@streamd/docs    # docusaurus on http://localhost:3000
```

## Build

The docs build is orchestrated by Turbo from the repo root:

```bash
npx turbo run build --filter=@streamd/docs
# output: apps/docs/build/
```

Or drive Docusaurus directly:

```bash
npm run build --workspace=@streamd/docs
```

## Deploy

The site deploys via the repo's release workflow — see
`.github/workflows/release.yml`. Preview a production build locally
with:

```bash
npm run serve --workspace=@streamd/docs
```

## Authoring

- Guides and recipes live under `docs/` as Markdown / MDX files.
  Each file's `sidebar_position` front-matter controls its order
  inside its category.
- Internal links between pages use relative paths (for example
  `../packages/parser`) so they resolve the same way locally, in
  production, and during Docusaurus' broken-link checker.
- API reference under `static/api/` is generated from TypeDoc — do
  not hand-edit.

This app intentionally does not depend on the `@streamd/*` runtime
packages. Docusaurus ships its own React runtime and any attempt to
share versions tends to create dual-React issues at build time.
