# @streamd/e2e

Internal — not published. End-to-end tests that cut across the
parser + renderers + plugins:

- **react-html-parity** — the React server-rendered markup matches the
  HTML renderer for the same token tree.
- **streaming-equivalence** — char-by-char streaming produces the same
  final HTML as a one-shot parse.
- **streaming-invariants** — stable-prefix and idempotence invariants
  hold across chunked inputs.
- **renderer-equivalence** — HTML and React renderers produce the same
  logical structure for the same token tree.
- **theming** — the CSS stylesheet generated by `renderThemeStylesheet`
  covers every class the HTML renderer emits.
- **plugins** — all five built-in plugins work through each renderer.
- **react-native-sample** — the RN renderer produces the expected
  primitive tree for the shared sample document.
- **public-api / public-types** — the surface exported by each package
  matches the declared contract.

## Run

```bash
npm run test
```

The suite asserts behavioural invariants rather than counts, so adding
a new integration case does not require updating this README.
